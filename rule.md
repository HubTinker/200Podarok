у меня есть задача: нужно подобрать 200 товаров и 50 впечатлений, вот по таким условиям: 
Цель задания
В ответ на мой запрос подготовить два JSON-файла:

new_gifts.json — список подарков (товары);
new_emotions.json — список впечатлений (услуги/опыты).

Каждый элемент — объект с полями: "name", "price", "purchaseUrl", "imageUrl". Всё в кавычках, без лишних символов и запятых.

Правила для всех записей
Структура полей (строго, в кавычках):
"name": короткое человекопонятное название (без моделей/серий/размеров/технических кодов).
"price": целое число в рублях, без пробелов, запятых и символов валют (например: "4990").
"purchaseUrl": ссылка на карточку товара на Яндекс.Маркете с хорошим рейтингом и отзывами (если товара нет на Яндекс.Маркете — предпочесть официальный магазин/популярный маркетплейс).
"imageUrl": имя файла изображения (без расширения) — формат "newgift_XXX" или "newemotion_XXX" — нумерация трёхзначная, начиная с 001.
Формат изображений:
Картинка в хорошем качестве, без надписей и лишних предметов.
Соотношение примерно 1:1, минимум 720px по меньшей стороне.
Файл сохраняется локально под именем, указанным в "imageUrl" (без расширения). Для каждого подарка/впечатления своё имя.
В поле "imageUrl" указывать только имя файла (например: "newgift_001").
Заполнение полей:
Всё содержимое внутри кавычек; не добавлять пробелы/символы вне кавычек.
Не дублировать уже существующие подарки (пользователь предоставил список существующих — сверять и избегать повторов).
Не включать бытовые, сугубо утилитарные или узконишевые предметы, которые плохо воспринимаются как подарок (сковороды, пылесосы, мебель, обычная одежда и т.п.).
Избегать сверхнише-ориентированных предметов, популярность только в узких сообществах.
Оценка подарочности:
При выборе учитывать, насколько предмет/активность подходит как подарок (не только полезность).
Подбирать разнообразные тематики и аудитории: женщины, мужчины, музыканты, геймеры, взрослые, молодые и т.д.
Правила для new_emotions.json (впечатления)
Тот же формат полей: "name", "price", "purchaseUrl", "imageUrl".
"purchaseUrl": сайт или активная страница организатора (предпочтительнее сайт).
"price": средняя цена в рублях (целое число).
"imageUrl": имя файла по шаблону "newemotion_XXX" (трёхзначная нумерация).
Нумерация файлов изображений
Подарки: newgift_001, newgift_002, ...
Впечатления: newemotion_001, newemotion_002, ...
Начинать нумерацию с 001 для каждого файла, не перескакивать и не дублировать имена.
Проверки перед финалом
Все строки в JSON — в двойных кавычках.
Цены — только числа (целые) внутри кавычек.
purchaseUrl — рабочая ссылка; у товаров — желателен Яндекс.Маркет с хорошими рейтингами.
imageUrl — корректное имя файла (без расширения), совпадающее с локальным файлом изображения.
Нет дублирования по name или по imageUrl с уже имеющимися товарами и товарами перечисленными в файле est.md.
Включены подарки разной тематики и для разных аудиторий.
Выходной результат (конкретно)
new_gifts.json — массив объектов: [{"name":"...","price":"...","purchaseUrl":"...","imageUrl":"newgift_001"}, ...]
new_emotions.json — массив объектов: [{"name":"...","price":"...","purchaseUrl":"...","imageUrl":"newemotion_001"}, ...]
Отдельная папка с изображениями, каждый файл назван согласно полю imageUrl (например newgift_001.jpg/png).
